test_LogInForm_get()[source]¶

test_loginForm() checks if the LogInForm is instantiated when an user access to login page. This test should be considered as different test with test_login_status_code() since this function just checks if the httpresponse returns the status_code 200.

test_LogInForm_post()[source]¶

test_LogInForm_post() tests if the LogINForm is properly instantiated with POST data. If the post data looks like {‘user_name’:’test_user’, ‘password’:’password’}, then the LogInForm must be called with the argument QueryDict(‘user_name=test_user&password=password’).

test_RegisterForm_invalid(*args, **kwargs)[source]¶

test_RegisterForm_validation() tests if it raise exception when the form is not valid. The attributes user_name and password are required. So if the registering user does not pass these two information, the form is not valid.
