User Authentication Documentation¶

Tests¶

Unit test for user_auth django application.

The list of tests are :

  • test_login_status_code() checks if accessing url /user_auth/login/ returns status_code 200.
  • test_register_status_code() checks if accessing url /user_auth/register/ returns status_code 200/
  • test_register_user() checks if RegisterForm.register_user() gets called when user_auth.views.register gets POST data.

setUp()[source]¶

setUp method sets up all neccessary modules. djanfo.TestCase inherits unittest so it follows same syntax that the unittest libary use.

tearDown()[source]¶

teatDown method removes all the changes that the test cases made for our system. This will be executed each time that the test case runs.

test_LogInForm_get()[source]¶

test_loginForm() checks if the LogInForm is instantiated when an user access to login page. This test should be considered as different test with test_login_status_code() since this function just checks if the httpresponse returns the status_code 200.

test_LogInForm_post()[source]¶

test_LogInForm_post() tests if the LogINForm is properly instantiated with POST data. If the post data looks like {‘user_name’:’test_user’, ‘password’:’password’}, then the LogInForm must be called with the argument QueryDict(‘user_name=test_user&password=password’).

test_RegisterForm_get()[source]¶

test_RegisterForm() checks when request.method to user_auth.views.register() is GET, the RegisterForm is properly instantiated.

test_RegisterForm_invalid(*args, **kwargs)[source]¶

test_RegisterForm_validation() tests if it raise exception when the form is not valid. The attributes user_name and password are required. So if the registering user does not pass these two information, the form is not valid.

test_RegisterForm_post()[source]¶

test_RegisterForm() checks when request.method to user_auth.views.register() is POST, the RegisterForm is properly instantiated with appropriate data. This test only checks if the RegisterForm is called with the POST data passed into register().

test_edit_EditForm_get()[source]¶

test_edit_EditForm_get() tests if the EditForm is properly instantiated when request.method is GET method.

test_edit_status_code()[source]¶

test_edit_status_code() checks if a user access to the edit page, the status_code equals to 200.

test_login_status_code()[source]¶

test_login_status_code() checks if a user access to the login page, the status_code is 200.

test_login_user_invalid_username(*args, **kwargs)[source]¶

test_login_user_invalid_username() tests if an user tries to login with invalid username such as non-exist username.

test_profile_not_logged_in()[source]¶

test_profile_not_logged_in() tests the status code of http response when the user who tries to access the profile page is not logged in. The status code should not be 200.

test_profile_status_code()[source]¶

test_profile_status_code() checks if a user access to the profile page, the status_code is 200.

test_register_notUnique_username(*args, **kwargs)[source]¶

test_register_notUnique_username() tests if a user trying to register with an username that is already exists in the database. This test should be failed since our website not allow two users with same username.

test_register_status_code()[source]¶

test_register_status_code() checks if a user access to the register page, the status_code is 200.

Model¶

This user_auth.models defined several user information.

  1. UserInfo defines those neccessary user information such as the date when each user registered, picture, expert_level and area of expertises.
  2. FriendList defines the list of friend for each user. Each user can have many friends.
